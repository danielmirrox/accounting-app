# Постановка задачи

Целью данной работы является закрепление навыков, полученных в течение всех практических работ. Разработано приложение на языке **Java** в соответствии с темой управления бухгалтерией компании.

Задача включала:
- Создание кластера MongoDB с использованием репликации
- Разработка полнофункционального приложения для управления данными
- Реализация операций CRUD (Create, Read, Update, Delete)
- Построение интерактивного пользовательского интерфейса
- Сортировка и поиск данных

# Практическая часть

## Архитектура приложения

Приложение разработано с использованием многоуровневой архитектуры:

```
src/main/java/
├── db/                          # Уровень доступа к данным
│   └── MongoDBConnection.java   # Подключение к MongoDB
├── models/                      # POJO классы
│   ├── Employee.java
│   ├── Salary.java
│   ├── Department.java
│   ├── Document.java
│   └── Report.java
├── crud/                        # CRUD операции
│   ├── EmployeeCRUD.java
│   ├── SalaryCRUD.java
│   ├── DepartmentCRUD.java
│   ├── DocumentCRUD.java
│   └── ReportCRUD.java
├── ui/
│   └── Menu.java                # Интерактивное меню
└── utils/
    ├── PrettyPrinter.java       # Форматирование вывода
    └── DataInitializer.java     # Инициализация данных
```

## Настройка MongoDB кластера

MongoDB кластер развёрнут в среде Docker с использованием docker-compose. Кластер состоит из 3 узлов:

- **mongo1**: localhost:30001 (primary)
- **mongo2**: localhost:30002 (secondary)  
- **mongo3**: localhost:30003 (secondary)

Репликация настроена с использованием Replica Set `my-replica-set`.

```yaml docker-compose.yml
```

## Модели данных

Приложение работает с пятью основными коллекциями:

### Employee (Сотрудники)

```java models/Employee.java
```

Хранит информацию о сотрудниках: ФИО, должность, паспортные данные.

### Salary (Зарплаты)

```java models/Salary.java
```

Связана с Employee через поле `employeeId`. Содержит месяц и сумму зарплаты.

### Department (Отделы)

```java models/Department.java
```

Информация об отделах и их руководителях.

### Document (Документы)

```java models/Document.java
```

Различные типы документов (накладные, акты, счета) с ответственными лицами.

### Report (Отчёты)

```java models/Report.java
```

Отчёты различных типов за определённые периоды.

## Реализация CRUD операций

Все CRUD операции реализованы в пакете `crud` с использованием MongoDB Java Driver и POJO Codec Registry для автоматической сериализации.

### EmployeeCRUD

```java crud/EmployeeCRUD.java
```

Методы для работы с сотрудниками:
- `addEmployee()` - добавление нового сотрудника
- `listEmployees()` - получение всех сотрудников
- `deleteEmployee()` - удаление сотрудника по ID
- `findEmployeeByName()` - поиск по имени (регулярное выражение, case-insensitive)
- `sortEmployeesBy()` - сортировка по полям

### SalaryCRUD

```java crud/SalaryCRUD.java
```

Аналогичные методы для работы с зарплатами.

### DepartmentCRUD

```java crud/DepartmentCRUD.java
```

Управление отделами.

### DocumentCRUD

```java crud/DocumentCRUD.java
```

Управление документами.

### ReportCRUD

```java crud/ReportCRUD.java
```

Управление отчётами.

## Подключение к базе данных

```java db/MongoDBConnection.java
```

Класс `MongoDBConnection` обеспечивает единую точку доступа к MongoDB базе данных "accounting" с поддержкой параметра `directConnection=true` для работы с Replica Set.

## Интерактивное меню

```java ui/Menu.java
```

Приложение предоставляет иерархическое консольное меню:

1. **Сотрудники** - добавить, показать, найти по имени, удалить, сортировать
2. **Зарплаты** - добавить, показать, удалить, сортировать
3. **Отделы** - добавить, показать, удалить, сортировать
4. **Документы** - добавить, показать, удалить
5. **Отчёты** - добавить, показать, удалить

## Форматирование и вывод данных

```java utils/PrettyPrinter.java
```

Класс `PrettyPrinter` обеспечивает красивый форматированный вывод всех данных с разделителями и структурированием.

## Инициализация тестовых данных

```java utils/DataInitializer.java
```

При запуске приложение предлагает инициализировать базу данных тестовыми данными:
- 3 сотрудника с паспортными данными
- 4 записи о зарплате
- 3 отдела
- 3 документа
- 3 отчёта

## Главный класс приложения

```java Main.java
```

Класс `Main` выполняет следующие функции:
- Подключение к MongoDB
- Предложение инициализировать данные
- Запуск интерактивного меню
- Корректное закрытие соединения с БД

## Зависимости и конфигурация

```xml pom.xml
```

Проект использует Maven для управления зависимостями:
- **MongoDB Java Driver 4.11.1** - синхронный драйвер для работы с MongoDB
- **Lombok 1.18.30** - сокращение boilerplate кода через аннотации

Проект настроен на Java 11+ с использованием Maven 3.6+.
